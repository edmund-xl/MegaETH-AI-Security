# 交接文档
<!-- security-log-analysis mainline -->

## 1. 目的

本文档用于在新机器、新 Codex 会话或新协作者接手时，快速恢复当前上下文，而不依赖旧对话记录。

## 2. 当前唯一产品域

当前仓库只服务于：

- 安全日志分析

任何已经废弃的实验产品线都不应重新引入。

## 3. 当前产品形态

当前系统应被理解为：

- 一个可训练、可复核、可持续校准的安全日志分析工作台

不应被理解为：

- 多产品平台
- 自动攻击或处置系统
- 第二独立控制面

## 4. 当前页面

有效页面只有五个：

- 概览
- 输入
- 技能
- 连接
- 学习

## 5. 当前工作模式

继续工作时，应遵循以下顺序：

1. 接收样本文件
2. 接收目标输出示例
3. 对齐分类与 Skill 路由
4. 对齐报告结构与判断边界
5. 回写案例文档、Skill 文档、测试和 GitHub

## 6. 已知教训

- 不要用其他项目文件复现当前项目问题
- 大文件不能只验证能否解析，还要验证表头、行数和聚合是否准确
- 新输入前要清理旧原始输入状态
- 浏览器端不要保存大块原始输入到 local storage
- 当用户只要求改一个模块时，不能顺手改其他模块

## 7. 新机器续接方式

推荐顺序：

1. clone 仓库
2. 阅读本文件
3. 阅读 README
4. 启动服务
5. 用真实样本做一次烟测
