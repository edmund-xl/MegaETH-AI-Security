# 训练工作流
<!-- security-log-analysis mainline -->

## 1. 目的

本文档描述如何基于真实样本持续训练和校准安全日志分析主线。

## 2. 训练输入

推荐一次训练至少具备以下信息：

- 原始样本文件
- 目标输出示例
- 期望分类
- 风险判断边界
- 报告结构要求

## 3. 标准训练流程

### 3.1 样本进入

- 上传样本文件或导入平台材料
- 确认输入是否完整
- 确认同批次材料是否应被联合判断

### 3.2 分类与 Skill 路由校准

- 检查 `source_type`
- 检查 `event_type`
- 检查命中的 Skill 是否正确

### 3.3 报告结构校准

- 页面版是否符合目标结构
- 下载版是否与页面版一致
- 结论是否符合目标样本的边界与口径

### 3.4 学习沉淀

- 将案例写入 `training_cases/`
- 更新对应 `skill_specs/`
- 更新必要的测试和说明文档

## 4. 当前重点训练主题

- Host baseline
- JumpServer 多源审计
- Whitebox AppSec 三段式

## 5. 训练产出要求

每次有效训练至少应带来以下变更中的一项或多项：

- 分类修正
- Skill 路由修正
- 报告模板修正
- 下载内容修正
- 学习反馈更新
- 案例文档更新

## 6. 注意事项

- 不要只验证“能解析”，还要验证表头、行数和聚合是否准确
- 不要让预览层裁剪污染分析层
- 不要让跨项目材料进入当前项目运行历史
- 修改共享链路后，应回归检查页面展示、下载与落库的一致性
